## 1. 최초 1회 — Bedrock 모델 접근 활성화

### 언제 필요한가
모든 파트 실행 전. 처음 Bedrock를 사용하는 계정이라면 모델 접근이 비활성화 상태입니다.

### 설정 방법
1. AWS 콘솔 → **Amazon Bedrock** → 왼쪽 메뉴 **Model access**
2. **Manage model access** 클릭
3. 아래 모델 체크 후 **Request model access** 클릭

| 모델 | 용도 |
|------|------|
| Anthropic — Claude Sonnet 4.6 | P2~P7 메인 모델 |
| Anthropic — Claude Haiku 4.5 | P7 자비스 AI 경량 모델 |
| Amazon — Nova Lite / Pro | P2 Prompt Routing |
| Amazon — Titan Text Embeddings V2 | P3 텍스트 KB 임베딩 |
| Amazon — Nova Multimodal Embeddings | P3 멀티모달 KB 임베딩 |

> 승인은 보통 즉시~수 분 이내 완료됩니다.

## 2. P2 — Cross-region Inference 활성화

### 언제 필요한가
`global.anthropic.claude-sonnet-4-6` 처럼 `global.` 접두사 모델 ID를 사용하는 모든 파트.

### 설정 방법
1. AWS 콘솔 → **Amazon Bedrock** → **Cross-region inference**
2. **Create inference profile** 버튼 클릭 (이미 있으면 생략)
3. 리전 프로파일 목록에 `global.anthropic.claude-sonnet-4-6` 이 보이면 활성화 완료

> 활성화 없이 실행하면 `ResourceNotFoundException` 오류 발생

## 3. P3 — Knowledge Base 생성 (가장 주의 필요)

### 왜 코드로 자동화할 수 없는가
SageMaker 실행 역할에는 아래 두 권한이 없어 콘솔 KB 생성 마법사만 자동 처리 가능합니다.
- `iam:CreateRole` — KB 서비스 역할 생성 불가
- `aoss:CreateCollection` — OpenSearch Serverless 컬렉션 생성 불가

### 실행 순서

```
python3 P3_02_rag_s3.py   ← 코드가 S3 버킷 + PDF 업로드 자동 처리
         ↓
★ 사용자: 콘솔에서 KB 2개 직접 생성 ★   ← 이 단계만 수동
         ↓
python3 P3_03_rag_kb_create.py --wait   ← 코드가 Sync + ID 저장 자동 처리
```

---

### 3-A. 텍스트 KB 생성 (telco-rateplan-text-kb)

1. AWS 콘솔 → **Amazon Bedrock** → **Knowledge bases** → **Create**
2. **Knowledge base type** 선택 화면에서 반드시:
   > ⚠️ **"Unstructured Vector Store KB"** 선택 (기본값 확인)
   > — "Structured" 또는 다른 유형 선택 시 이후 retrieve API가 동작하지 않음

3. **이름**: `telco-rateplan-text-kb`
4. **IAM**: `Create and use a new service role` 선택 (새 역할 자동 생성)
5. **데이터 소스** → **Amazon S3** 선택
   - S3 URI: `s3://telco-rateplan-kb-2026-<계정뒤4자리>/text/`
   - ⚠️ URI 끝에 `/` 슬래시 반드시 포함
6. **파서**: `Default parser` 선택
7. **임베딩 모델**: `Titan Text Embeddings V2` (`amazon.titan-embed-text-v2:0`)
8. **벡터 DB**: `Amazon OpenSearch Serverless` → **Quick create** 선택
   > ⚠️ **"Quick create"** 선택 필수 — "Custom" 선택 시 별도 컬렉션 사전 생성 필요
9. **Create knowledge base** 클릭 → 생성까지 약 3~5분 소요

---

### 3-B. 멀티모달 KB 생성 (telco-rateplan-multimodal-kb)

1. **Create knowledge base** → **Unstructured Vector Store KB** 선택
2. **이름**: `telco-rateplan-multimodal-kb`
3. **IAM**: `Create and use a new service role`
4. **데이터 소스** → **Amazon S3**
   - S3 URI: `s3://telco-rateplan-kb-2026-<계정뒤4자리>/multimodal/`
5. **파서**: `Bedrock Data Automation (BDA)` 선택
   > ⚠️ Default parser가 아닌 **BDA** 선택 — 이미지/표/그래프 이해에 필수
6. **MM output 버킷**: `s3://telco-rateplan-kb-mm-output-2026-<계정뒤4자리>/`
   > ⚠️ 데이터 버킷과 다른 output 전용 버킷 지정 (P3_02에서 자동 생성됨)
7. **임베딩 모델**: `Amazon Nova Multimodal Embeddings` (`amazon.nova-2-multimodal-embeddings-v1:0`)
   > ⚠️ 텍스트 KB와 다른 임베딩 모델 선택 — 벡터 차원 3072 (텍스트는 1024)
8. **벡터 DB**: `Amazon OpenSearch Serverless` → **Quick create**
9. **Create knowledge base** 클릭

---

### 3-C. KB ID 확인 및 기록

KB 생성 완료 후 각 KB의 **ID** (예: `ABCDE12345`) 를 메모합니다.
`P3_03_rag_kb_create.py` 실행 시 자동으로 ID를 탐지하여 `rag_config.json`에 저장합니다.

---

### 3-D. OpenSearch 데이터 접근 정책 (P3_06 오류 발생 시)

`P3_06_rag_opensearch.py` 실행 중 HTTP 403 오류가 발생하면:

1. AWS 콘솔 → **Amazon OpenSearch Service** → **Serverless** → **Collections**
2. KB가 생성한 컬렉션 클릭 → **Data access** 탭
3. 기존 정책 **편집** → **Principals** 에 아래 ARN 추가:
   ```
   arn:aws:iam::<계정ID>:role/AmazonSageMaker-ExecutionRole-*
   ```
4. 저장

## 4. P4 — Guardrails 사전 설정

### 4-A. Cross-region Inference Standard Tier 활성화

Guardrails Standard Tier는 cross-region inference가 활성화된 상태에서만 사용 가능합니다.

1. AWS 콘솔 → **Amazon Bedrock** → **Cross-region inference**
2. Standard Tier 프로파일이 존재하는지 확인
3. 없으면 **Create inference profile** 클릭 후 생성

> 미활성화 시 `P4_04_guardrails_create.py` 실행 중 Guardrail 생성 단계에서 오류 발생

### 4-B. Guardrail 생성

P4는 `P4_04_guardrails_create.py`가 **코드로 자동 생성**합니다.
별도 콘솔 작업 없이 아래 명령으로 생성 후 `guardrail_config.json`에 자동 저장됩니다.

```bash
python3 P4_04_guardrails_create.py
```

이후 P4_05~P4_07은 이 파일을 자동으로 읽어 실행합니다.

## 5. P7 — 음성 기능 IAM 권한 추가

### 언제 필요한가
`P7_01_jarvis_stt_tts.py` 또는 `P7_02_jarvis_streamlit.py`의 TTS 기능 사용 시.
권한 없이 실행하면 자동으로 안내 메시지와 IAM 정책 JSON이 출력됩니다.

### 추가할 IAM 정책 JSON

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "JarvisPollyAccess",
      "Effect": "Allow",
      "Action": [
        "polly:SynthesizeSpeech",
        "polly:DescribeVoices"
      ],
      "Resource": "*"
    },
    {
      "Sid": "JarvisTranscribeAccess",
      "Effect": "Allow",
      "Action": [
        "transcribe:StartTranscriptionJob",
        "transcribe:GetTranscriptionJob",
        "transcribe:ListTranscriptionJobs",
        "transcribe:DeleteTranscriptionJob"
      ],
      "Resource": "*"
    },
    {
      "Sid": "JarvisS3Access",
      "Effect": "Allow",
      "Action": [
        "s3:CreateBucket",
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject",
        "s3:ListBucket",
        "s3:HeadBucket"
      ],
      "Resource": [
        "arn:aws:s3:::jarvis-stt-tts-*",
        "arn:aws:s3:::jarvis-stt-tts-*/*"
      ]
    }
  ]
}
```

자세한 추가 방법은 `docs/SageMaker_IAM_권한설정.md` 참고.

## 6. Config 파일 의존성 구조

코드가 자동 생성하는 config 파일과 이를 참조하는 파일 관계:

```
P3_02_rag_s3.py
  └─ S3 버킷 생성 (자동)

P3_03_rag_kb_create.py
  └─ rag_config.json 생성 (자동) ──┐
                                    ├── P3_04_rag_kb_test.py
                                    ├── P3_05_rag_retrieve.py
                                    ├── P3_06_rag_opensearch.py
                                    ├── P3_07_rag_chatbot.py
                                    └── P3_08_rag_managed_kb.py

P4_04_guardrails_create.py
  └─ guardrail_config.json 생성 (자동) ──┐
                                          ├── P4_05_guardrails_converse.py
                                          ├── P4_06_guardrails_apply.py
                                          ├── P4_07_guardrails_chatbot.py
                                          └── P4_05_guardrails_converse_streamlit.py
```

> ⚠️ config 파일이 없는 상태에서 의존 파일을 실행하면 즉시 오류 종료됩니다.
> 항상 위 순서대로 실행하세요.

## 7. 전체 실행 순서 요약

```
[최초 1회]
  → Bedrock 모델 접근 활성화 (콘솔)
  → Cross-region inference 활성화 (콘솔)

[P2]
  python3 P2_01 ~ P2_05    ← 전부 자동
  streamlit run P2_06       ← 자동

[P3]
  python3 P3_01             ← 권한 점검 자동
  python3 P3_02             ← S3 버킷 + 업로드 자동
  ★ 콘솔에서 KB 2개 생성 ★  ← 사용자 직접 (약 5~10분)
  python3 P3_03 --wait      ← Sync + 설정 저장 자동
  python3 P3_04 ~ P3_06     ← 전부 자동
  streamlit run P3_07       ← 자동
  python3 P3_08 --delete-all ← 실습 완료 후 정리 (AOSS 비용 방지)

[P4]
  python3 P4_01             ← 권한 점검 자동
  python3 P4_04             ← Guardrail 생성 자동
  python3 P4_05 ~ P4_06     ← 전부 자동
  streamlit run P4_07       ← 자동

[P5]
  python3 P5_01 ~ P5_06     ← 전부 자동

[P6]
  python3 P6_01 ~ P6_11     ← 전부 자동
  streamlit run P6_10_...   ← 자동

[P7]
  ★ IAM 역할에 Polly/Transcribe 권한 추가 ★  ← 사용자 직접
    (docs/SageMaker_IAM_권한설정.md 참고)
  python3 P7_01             ← STT/TTS 검증 자동
  streamlit run P7_02       ← 자비스 챗봇 자동
  python3 P7_03             ← Nova Sonic 안내 자동
```

## 8. 자주 발생하는 오류와 원인

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|----------|
| `ResourceNotFoundException` (global. 모델) | Cross-region 프로파일 미활성화 | 2번 항목 — 콘솔에서 활성화 |
| `rag_config.json 없음` | P3_03 미실행 또는 KB 미생성 | KB 생성 후 P3_03 재실행 |
| `guardrail_config.json 없음` | P4_04 미실행 | `python3 P4_04_guardrails_create.py` |
| `AccessDeniedException` (Polly/Transcribe) | IAM 역할에 권한 없음 | 5번 항목 — IAM 정책 추가 |
| `HTTP 403` (OpenSearch) | AOSS 데이터 접근 정책 미등록 | 3-D 항목 — 정책에 역할 ARN 추가 |
| `KB 상태 CREATING` | KB 생성 중 (정상) | `P3_03 --wait` 옵션으로 자동 대기 |
| `iam:CreateRole 없음` | 코드로 KB 자동 생성 시도 | 콘솔에서 직접 KB 생성 (3번 항목) |

## 9. 비용 주의사항

| 서비스 | 주의 사항 | 조치 |
|--------|----------|------|
| **OpenSearch Serverless** | 컬렉션이 존재하는 동안 OCU 비용 발생 (사용 안 해도 과금) | 실습 완료 후 `P3_08 --delete-all` 실행 |
| **S3** | 버킷 삭제하지 않으면 소량의 스토리지 비용 지속 발생 | `P3_08 --delete-s3` 로 함께 정리 |
| **Bedrock API** | API 호출마다 토큰 비용 발생 (교육 수준은 미미) | 특이사항 없음 |
| **Transcribe** | 오디오 분당 과금 (실습 수준은 미미) | 특이사항 없음 |
| **Polly** | 문자 수 기준 과금 (실습 수준은 미미) | 특이사항 없음 |
