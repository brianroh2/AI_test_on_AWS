# P7 — JARVIS 코드 진화 가이드

이 문서는 P7 시리즈의 코드가 파일을 거치면서 어떻게 발전하는지 설명합니다.  
P6까지는 텍스트 기반 Agent였다면, P7은 **음성**을 더해 진짜 AI 비서로 완성됩니다.

---

## P6 → P7 전환 핵심

```
P6: 텍스트 입력 → Strands Agent → 텍스트 출력
         ↓
P7: 텍스트 입력 → Strands Agent → 텍스트 출력 + 음성(Polly) 출력
    (미래)  음성 입력(Transcribe) → Agent → 음성 출력
```

---

## 전체 진화 한눈에 보기

```
P7_01          →   P7_02              →   P7_04
Polly/STT 검증     JARVIS 기본 챗봇       JARVIS 고도화

                    P7_03

                    (비교 참고용)
```

## 1단계 — P7_01: 부품 단독 검증

**목적:** Polly(TTS)와 Transcribe(STT)를 조립하기 전에 각각 동작하는지 확인합니다.

### 검증 항목

```
┌─────────────────────────────────────────────────────────┐
│  1. IAM 권한 사전 점검                                   │
│     polly:SynthesizeSpeech                              │
│     transcribe:StartTranscriptionJob                    │
│     → 권한 없으면 필요한 IAM 정책 JSON 자동 출력        │
│                                                         │
│  2. Polly TTS 동작 확인                                  │
│     텍스트 → MP3 파일 생성 → /tmp/jarvis_test.mp3       │
│                                                         │
│  3. Transcribe STT 동작 확인                            │
│     MP3 → S3 업로드 → 변환 작업 시작 → 텍스트 반환     │
│     (비동기: 작업 완료까지 폴링 대기)                    │
└─────────────────────────────────────────────────────────┘
```

### Polly TTS 핵심 코드

```python
polly = boto3.client("polly", region_name="us-east-1")

resp = polly.synthesize_speech(
    Text="안녕하세요, 저는 자비스입니다.",
    OutputFormat="mp3",
    VoiceId="Seoyeon",      # 한국어 Neural 음성
    Engine="neural",        # Neural 엔진 필수 (Standard는 품질 낮음)
    LanguageCode="ko-KR",
)
audio_bytes = resp["AudioStream"].read()   # bytes → 파일 저장 또는 브라우저 재생
```

```
Polly 3가지 필수 파라미터:
  VoiceId      → 음성 선택 (언어별로 다름)
  Engine       → "neural" 고정 (standard보다 훨씬 자연스러움)
  LanguageCode → VoiceId와 반드시 일치해야 함
                 불일치 시 InvalidSampleRateException 발생
```

### Transcribe STT 핵심 코드

```python
transcribe = boto3.client("transcribe", region_name="us-east-1")

# 1. 작업 시작 (비동기)
transcribe.start_transcription_job(
    TranscriptionJobName="jarvis-test-001",
    Media={"MediaFileUri": f"s3://버킷명/test.mp3"},
    MediaFormat="mp3",
    LanguageCode="ko-KR",
)

# 2. 완료 대기 (폴링)
while True:
    status = transcribe.get_transcription_job(TranscriptionJobName="jarvis-test-001")
    if status["TranscriptionJob"]["TranscriptionJobStatus"] in ["COMPLETED", "FAILED"]:
        break
    time.sleep(5)

# 3. 결과 가져오기
transcript_uri = status["TranscriptionJob"]["Transcript"]["TranscriptFileUri"]
```

```
⚠️ Transcribe는 동기가 아님 → 완료까지 수 초~수십 초 대기 필요
⚠️ 음성 파일을 S3에 먼저 업로드해야 함 (로컬 파일 직접 처리 불가)
⚠️ SageMaker 기본 역할에 Polly/Transcribe 권한 없음 → IAM 수동 추가 필요
```

## 2단계 — P7_02: JARVIS 기본 챗봇 — P6 Agent + TTS 결합

**목적:** P6의 Strands Agent에 Polly TTS를 붙여 텍스트+음성으로 동시에 응답합니다.

### 파이프라인

```
┌─────────────────────────────────────────────────────────┐
│  사용자 입력 (텍스트)                                    │
│       ↓                                                 │
│  Strands Agent (Haiku 모델)                             │
│    tools: weather_forecast, web_search, calculator      │
│    system_prompt: "3문장 이내로 답변" ← 음성 최적화      │
│       ↓                                                 │
│  텍스트 응답                                            │
│       ↓                           ↓                     │
│  화면에 텍스트 표시          Polly TTS → MP3 → 자동 재생 │
└─────────────────────────────────────────────────────────┘
```

### 핵심 코드

```python
# TTS 함수: 마크다운 기호/URL 제거 후 Polly 호출
def text_to_speech(text: str) -> bytes | None:
    clean = re.sub(r'[*_`#\[\]()]', '', text)      # 마크다운 제거
    clean = re.sub(r'https?://\S+', '', clean)      # URL 제거
    resp  = polly_client.synthesize_speech(
        Text=clean[:3000],       # Polly 최대 3000자 제한
        OutputFormat="mp3",
        VoiceId="Seoyeon",
        Engine="neural",
        LanguageCode="ko-KR",
    )
    return resp["AudioStream"].read()

# 브라우저 자동 재생 (HTML audio 태그)
def autoplay_audio(audio_bytes: bytes):
    b64 = base64.b64encode(audio_bytes).decode()
    st.markdown(
        f'<audio autoplay style="display:none;">'
        f'<source src="data:audio/mp3;base64,{b64}" type="audio/mp3"></audio>',
        unsafe_allow_html=True,
    )

# Agent 호출 → TTS 재생
result = agent(user_input)
text   = str(result).strip()
audio  = text_to_speech(text)
if audio:
    autoplay_audio(audio)
```

### 설계 포인트

```
1. 모델: Haiku 선택 이유
   → 음성 대화는 긴 답변이 불필요
   → Haiku가 Sonnet 대비 응답 속도 3배 빠름 + 비용 1/10
   → 사용자가 사이드바에서 Sonnet으로 전환 가능

2. system_prompt: "3문장 이내"
   → 음성으로 듣기에 3문장이 적당
   → 길면 TTS 재생 시간이 길어져 대화감이 떨어짐

3. TTS 전처리 필수
   → Polly는 마크다운(*,#,_)을 그대로 읽음
   → "**서울**" → "별별서울별별" 처럼 읽힘
   → 전처리로 특수문자 제거 후 전달
```

### 추가 기능 3: 문서 RAG (InlineDocumentSource)

```python
# PDF/TXT 업로드 → 텍스트 추출 → 질문과 관련된 청크만 선택
def build_rag_context(question: str, doc_texts: list[str]) -> str:
    keywords = set(question.split())       # 질문 키워드 추출
    scored = []
    for doc in doc_texts:
        chunks = [doc[i:i+500] for i in range(0, len(doc), 400)]   # 500자 청크
        for chunk in chunks:
            score = sum(1 for kw in keywords if kw in chunk)        # 키워드 매칭
            if score > 0:
                scored.append((score, chunk))
    scored.sort(reverse=True)
    return "\n---\n".join(c for _, c in scored[:4])   # 상위 4개 청크만 사용

# 질문 앞에 문서 컨텍스트를 붙여서 Agent에 전달
if rag_ctx:
    full_q = f"[문서 컨텍스트]\n{rag_ctx}\n\n질문: {question}"
```

## 학습 포인트 정리

```
1. Polly 3대 주의사항
   → Engine="neural" 필수 (standard는 로봇 목소리)
   → LanguageCode와 VoiceId 반드시 일치
   → 마크다운/URL 전처리 후 전달 (안 하면 기호를 읽어버림)

2. SSML 속도 조절 시 TextType="ssml" 추가 필수
   → 빠뜨리면 <speak> 태그 자체를 읽어버림

3. Transcribe는 비동기
   → 즉시 결과를 못 받음, 폴링 루프 필요
   → 음성 파일을 S3에 먼저 올려야 함

4. 음성 최적화를 위한 시스템 프롬프트 전략
   → "3문장 이내" 지시: 음성으로 듣기에 적합한 길이
   → "URL, 이모지, 목록 형식 금지": TTS가 이상하게 읽는 요소 제거

5. RAG와 웹검색의 우선순위
   → 문서가 있으면 문서 우선
   → 문서에 없으면 web_search 도구로 자동 보완
   → build_rag_context()로 관련 청크만 추출 (전체 문서 전달 X)

6. 통역 모드는 Agent 아닌 converse() 직접 사용
   → 도구 호출이 불필요한 순수 번역 작업
   → Agent 오버헤드 없이 더 빠르게 응답
```
